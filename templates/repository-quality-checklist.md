# Repository Quality Checklist

## Required Files

- README.md
- ARCHITECTURE.md
- COURSE_MAP.md
- ROADMAP.md
- CONTRIBUTING.md
- CHANGELOG.md
- LICENSE

## Required Folders

- `/docs`
- `/modules`
- `/lessons`
- `/labs`
- `/projects`
- `/templates`
- `/governance`
- `/diagrams`
- `/assets`
- `/examples`
- `/capstone`
- `/assessments`
- `/rubrics`
- `/checklists`
- `/playbooks`
- `/troubleshooting`

## Course Metadata

- Course Title
- Repository Name
- Difficulty
- Estimated Duration
- Target Audience
- Business Value
- Technical Value
- Operational Value
- Portfolio Deliverables
- Capstone Outcome

## Module Quality

- learning goals are specific
- business context is enterprise-relevant
- technical context is implementation-oriented
- operational context includes ownership and workflow
- architecture discussion is present
- governance considerations are present
- each module includes a lab
- deliverables are explicit

## Lesson Quality

- lesson overview is clear
- business problem is specific
- technical context is concrete
- operational context identifies roles and impact
- walkthrough is actionable
- common failures are realistic
- troubleshooting is included
- best practices are implementation-ready
- governance and security are addressed
- validation steps are measurable
- expected outputs are explicit

## Lab Quality

- scenario is operationally realistic
- business problem is clear
- technical requirements are testable
- operational requirements are included
- architecture requirements are included
- validation checks are measurable
- troubleshooting paths are included
- portfolio deliverable is clear

## Capstone Quality

- enterprise simulation is specific
- operational challenge is realistic
- architecture review is required
- implementation validation is required
- governance review is required
- deployment model is production-style
- scoring rubric is included

## Public Repository Review

- only polished public-facing content
- no conversational approval language
- no unresolved placeholders outside templates
- no private credentials
- no customer confidential data
- no proprietary system exports
- no personal information
- no unsupported certification claims
